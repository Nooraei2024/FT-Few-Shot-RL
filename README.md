# Replication package: FT Few-Shot RL for Requirements Classification

This package accompanies the manuscript **“Benchmarking Few-Shot Reinforcement Learning and Human-in-the-Loop Techniques for Requirements Classification in Data-Scarce Settings.”**

The package is organized to support two forms of reproducibility:

1. **Reported-results reproduction.** The scripts read the Excel result tables extracted from the manuscript and regenerate the tables and figures used in the paper. This is the safest route for checking consistency with the reported manuscript values.
2. **Executable method scaffold.** The code also includes a fuller implementation of the experimental workflow: preprocessing, classical baselines, BERT fine-tuning hooks, prototypical few-shot classification, and reward-guided fine-tuning with expert-feedback weighting. This part requires the dataset to be downloaded separately.

## Dataset

The PROMISE dataset is not redistributed here. Download it from Kaggle:

https://www.kaggle.com/datasets/iamsouvik/promise

After downloading, place the CSV file in:

```text
data/promise_nfr.csv
```

If your filename is different, pass it using `--data`.

## Main files

```text
results/extracted_results_from_manuscript.xlsx   Reported results extracted from the manuscript
code/reproduce_reported_results.py               Regenerates tables and figures from the Excel file
code/check_consistency.py                         Checks that expected manuscript values are present
code/run_training_pipeline.py                     Training/evaluation scaffold for the method
code/method/ft_fewshot_rl.py                      Main method implementation
configs/experiment_config.json                    Experimental parameters
prompts/prompts_used.txt                          Prompt templates used in the study
outputs/                                          Tables and figures generated by reproduction scripts
```

## Recommended reviewer command

From the package root, run:

```bash
python code/reproduce_reported_results.py
python code/check_consistency.py
```

This will regenerate:

```text
outputs/tables/table3_performance.csv
outputs/tables/table4_data_ablation.csv
outputs/tables/component_ablation.csv
outputs/figures/fig2_performance_comparison.png
outputs/figures/fig3_data_ablation.png
outputs/figures/fig4_component_ablation_f1.png
outputs/figures/fig4_component_ablation_accuracy.png
```

## Running the method scaffold

After downloading the Kaggle dataset, run for classical baselines:

```bash
python code/run_training_pipeline.py --data data/promise_nfr.csv --model svm --task fr_nfr
python code/run_training_pipeline.py --data data/promise_nfr.csv --model logistic --task fr_nfr
```

The BERT and reward-guided few-shot variants require PyTorch and Transformers. Exact numerical reproduction of trained-model results can vary due to dataset version, preprocessing, random seeds, hardware, and library versions. The reported numerical values used in the manuscript are therefore also provided directly in the Excel file under `results/`.

## Transparency note

The package separates **reported-result reproduction** from **model re-execution**. The reported-result scripts reproduce the manuscript tables and figures exactly from the extracted result workbook. The training scripts are provided to document the experimental workflow and enable further verification when the dataset and dependencies are available.
